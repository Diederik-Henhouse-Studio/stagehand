## 🚨 KRITIEKE PROBLEMEN

### 1. **Dropdown Navigatie Faalt** ❌

**Geprobeerde methodes (alle faalden):**

#### Poging 1: `page.selectOption()` (Playwright)
```typescript
await page.selectOption('#CurrentYear', { value: '2025' });
```
**Error:** `TypeError: page.selectOption is not a function`
**Oorzaak:** Stagehand Page object heeft niet alle Playwright methodes

#### Poging 2: `stagehand.act()`
```typescript
await this.stagehand.act({ action: `select 2025 from year dropdown` });
```
**Error:** `StagehandInvalidArgumentError: instruction string is required`
**Oorzaak:** Verkeerde API syntax

#### Poging 3: DOM Manipulation
```typescript
await page.evaluate((y, m) => {
  yearSelect.value = y.toString();
  monthSelect.value = m.toString();
  // Trigger events
});
```
**Error:** `StagehandEvalError: Uncaught TypeError`
**Oorzaak:** Parameters worden niet correct doorgegeven

### 2. **URL Parameter Wordt Genegeerd** ❌

**Getest:**
```
https://oirschot.bestuurlijkeinformatie.nl/Calendar?year=2025&month=11
```

**Resultaat:**
- URL laadt succesvol ✅
- Maar toont **januari 2026 meetings** ❌

**Conclusie:**
Website gebruikt client-side JavaScript dat URL parameters negeert en altijd huidige/eerstvolgende maand laadt.

### 3. **Maand Index Bug (gefixed)** ✅ → ⚠️

**Origineel probleem:**
```typescript
// ❌ FOUT: month=12 geeft januari volgend jaar
const url = `Calendar?year=2025&month=12`;
```

**Fix:**
```typescript
// ✅ CORRECT: month - 1 voor 0-indexed
const monthIndex = month - 1; // December = 11
const url = `Calendar?year=2025&month=${monthIndex}`;
```

**Status:** Fix toegepast, maar URL parameters werken niet dus geen effect.

## 🎯 ROOT CAUSE ANALYSE

### Waarom Navigatie Faalt

**Hypothese 1: JavaScript Event Listeners**
De website gebruikt waarschijnlijk specifieke event listeners op de dropdowns die:
- Complexe AJAX requests doen
- Met specifieke parameters/tokens
- Die we niet kunnen repliceren

**Hypothese 2: Server-side Session State**
De kalender status wordt server-side bijgehouden en URL parameters zijn alleen hints, geen commands.

**Hypothese 3: Framework-specific Binding**
Als de site React/Vue/Angular gebruikt, heeft het waarschijnlijk framework-specific state management dat we niet kunnen triggeren via DOM events.

### Bewijs
```javascript
// Alle geprobeerde events:
['change', 'input', 'click'].forEach(event => {
  dropdown.dispatchEvent(new Event(event, { bubbles: true }));
});

// jQuery trigger (indien beschikbaar):
$(dropdown).trigger('change');

// Resultaat: Geen AJAX request, geen data update
```

## 🎓 LESSONS LEARNED

### 1. Stagehand API Limitations
- `page.selectOption()` niet beschikbaar
- `page.evaluate()` heeft issues met parameters
- `stagehand.act()` syntax onduidelijk
- Fallback op basis Playwright functies nodig

### 2. Website Behavior
- URL parameters zijn **hints**, geen **commands**
- Client-side JavaScript heeft authoriteit over data
- Dropdown interactie triggert complexe workflows
- Kan niet gesimuleerd worden met simpele events

### 3. IBABS Platform
- Consistent URL patroon: `/Agenda/Index/{uuid}`
- Gebruik van dropdowns voor navigatie
- Rijke metadata beschikbaar
- Grote hoeveelheid bijlagen per meeting
